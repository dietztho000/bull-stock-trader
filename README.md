# BULL Stock Trader — Autonomous Trading Bot

A cloud-scheduled, Git-as-memory trading agent built on Claude Code. **Ten
cron routines** fire each weekday: pre-dawn auth-canary, pre-market research,
market-open execution, mid/late-morning stop promotion, midday scan, stop
reconciliation, afternoon thesis check, daily summary, and Friday weekly
review. **Claude is the bot** — there is no separate Python process. Every
run is a fresh container that clones `main`, reads memory, calls the live
APIs, writes new memory, and pushes.

A Next.js dashboard at [`dashboard/`](dashboard/) sits alongside the cloud
agent, reading the same `memory/` tree and providing live-Alpaca tiles, a
multi-bot manager (`/bots`), backtest snapshots, calendar, and AI-assisted
post-mortems.

## Strategy at a glance
Swing trading stocks only — no options, ever. Max 5–6 positions, max 20% per
position, max 3 new trades per week, 75–85% deployed. Every position gets a
real 10% trailing stop GTC order on Alpaca. Cut losers at -7%; tighten the
trail to 7% at +15% and 5% at +20%. Exit a sector after two consecutive
failed trades. Patience > activity.

Full rules: `memory/<bot>/<strategy>/TRADING-STRATEGY.md` (default live bot:
[`memory/live/default/TRADING-STRATEGY.md`](memory/live/default/TRADING-STRATEGY.md)).

## Local quickstart

```bash
cp env.template .env
# fill in ALPACA_*, PERPLEXITY_*, DISCORD_WEBHOOK_URL values
chmod +x scripts/*.sh
bash scripts/install-githooks.sh   # one-time: enable routines/ round-trip pre-push
```

Then in Claude Code:

```
/portfolio       # read-only snapshot — smoke test
/benchmark       # YTD-vs-SPY snapshot — read-only
/auth-canary     # 5-min preflight ping to all 4 APIs (cron: weekdays 3:30am CT)
/pre-market      # write today's research
/market-open     # validate + execute
/mid-morning     # promote stops, escalate buys
/late-morning    # thesis check + stop promotion
/midday          # full intraday scan + Perplexity
/stops           # reconcile every position has the right trailing stop
/afternoon       # late-day thesis check
/daily-summary   # EOD recap + Discord message + run-log watchdog
/weekly-review   # Friday only
/trade SYM N buy # ad-hoc trade with full rule + entry-scorer validation
```

## Multi-bot

A "bot" is a named trading agent bound to one Alpaca account, optionally with
a soft capital allocation (e.g. carve a $10k slice off a $100k paper account
to test a new strategy without spinning up a separate broker account).

Manage bots in the dashboard at `/bots`:
- **Add account** — paste API key + secret + endpoint; the dashboard
  encrypts them with `BULL_VAULT_KEY` (AES-256-GCM) and stores them in
  `memory/shared/dashboard-settings.json` (gitignored).
- **Add bot** — name + bound account + optional allocation + strategy slug.
- Cloud routines fan out across enabled bots automatically (each STEP runs
  N times for N bots, with auth-preflight per account).

Memory tree per bot at `memory/<bot-id>/<strategy>/`. Cross-bot caches in
`memory/shared/`.

## Cloud setup

See [routines/README.md](routines/README.md) for the routine setup, the cron
schedule, and the env vars each routine needs. The two things that break
first-time setups every time:

1. The Claude GitHub App must be installed on this repo.
2. **"Allow unrestricted branch pushes"** must be toggled on in each routine's
   environment. Without it, `git push origin main` silently fails and
   `scripts/cron-sync.sh` (running locally every 15 min via launchd) is what
   keeps the dashboard's view of memory current.

## Memory model

Markdown files in `memory/<bot>/<strategy>/` are each bot's only state between
runs. Every write is an append-only dated section, so merge conflicts are
effectively impossible across the hours-apart routine schedule. Idempotency
is enforced via grep-first date anchors (see `CLAUDE.md` "Memory write
idempotency").

| File                      | Scope     | Cadence                                  |
|---------------------------|-----------|------------------------------------------|
| `TRADING-STRATEGY.md`     | Per-bot   | Friday only, when a rule changes         |
| `TRADE-LOG.md`            | Per-bot   | Every trade, every EOD                   |
| `RESEARCH-LOG.md`         | Per-bot   | Every pre-market                         |
| `BENCHMARK.md`            | Per-bot   | Every EOD (one row/day vs SPY)           |
| `SECTOR-LEDGER.md`        | Per-bot   | On every closed trade (W/L)              |
| `WEEKLY-REVIEW.md`        | Per-bot   | Fridays                                  |
| `RUN-LOG.jsonl`           | Per-bot   | Every routine start + end (heartbeat)    |
| `EARNINGS-CALENDAR.md`    | Per-bot   | Refreshed on holdings change             |
| `BACKTEST-RESULTS.{md,json}` | Per-bot | Dashboard's "Run fresh" button           |
| `SECTOR-MAP.md`           | Shared    | Cached GICS lookup per ticker            |
| `ECONOMIC-CALENDAR.md`    | Shared    | Daily refresh (FOMC, CPI, NFP, …)        |
| `MARKET-EARNINGS.md`      | Shared    | Cross-market earnings calendar           |
| `PERPLEXITY-LOG.md`       | Shared    | Every Perplexity query (cost telemetry); cache at `.perplexity-cache.jsonl` for fan-out idempotency |

## Editing workflow steps

The STEP content for each cloud routine lives in `.claude/commands/<name>.md`
between `<!-- STEPS-BEGIN -->` and `<!-- STEPS-END -->` markers — the single
source of truth. To regenerate the cloud routine prompts after editing a
command:

```bash
bash scripts/build-routines.sh
git add .claude/commands/<name>.md routines/<name>.md
```

The `pre-push` hook in `.githooks/pre-push` (installed by
`scripts/install-githooks.sh`) refuses to push when `routines/` is out of
sync with `.claude/commands/`.

## Notification philosophy

Most bots are chatty. This one isn't. Pre-market silent unless urgent.
Market-open only if a trade fires. Midday only if action was taken. The
daily summary and weekly review always send. The cost of a missed alert is
low (you can `/portfolio` anytime); the cost of a chatty bot is that you
stop reading.

For boring-tech redundancy, set `NTFY_TOPIC` to push notifications to your
phone via [ntfy.sh](https://ntfy.sh) alongside Discord — no extra credentials.
